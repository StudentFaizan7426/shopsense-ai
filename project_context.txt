TECH STACK:
- Python 3.11.7 (Anaconda)
- Flask + Flask-SQLAlchemy + Flask-CORS
- SQLite database (data/shop.db)
- scikit-learn (3 ML models)
- Groq API (llama-3.3-70b-versatile)
- Chart.js + Apache ECharts
- jsPDF (PDF generation)
- bcrypt (password hashing)
- Token-based authentication

FOLDER STRUCTURE:
backend/
  app.py           → Main Flask entry point
  database.py      → SQLAlchemy setup
  models.py        → All DB table definitions
  ml_models.py     → 3 ML prediction models
  .env             → GROQ_API_KEY stored here
  routes/
    products.py    → Product CRUD
    sales.py       → Sales + expenses + reports
    charts.py      → Chart data endpoints
    predictions.py → ML model endpoints
    chatbot.py     → Groq AI integration
    billing.py     → Cart/billing system
    auth.py        → Token-based login system

data/
  shop.db          → SQLite database file

frontend/          → Served through Flask
  login.html       → Login page (entry point)
  index.html       → Main dashboard
  products.html    → Product management
  sales.html       → Daily sales recording
  expenses.html    → Expense tracking
  reports.html     → PDF report download
  billing.html     → Cart/billing system
  admin.html       → Staff management
  error-handler.js → Global JS error handler

DATABASE TABLES (7):
1. products      → id, name, category,
                   purchase_price,
                   selling_price,
                   current_stock, reorder_level
2. daily_sales   → id, product_id(FK),
                   quantity_sold, sale_price,
                   revenue, date, notes
3. expenses      → id, category, amount,
                   date, description
4. stock_updates → id, product_id(FK),
                   quantity_added,
                   supplier_price, date
5. bills         → id, bill_number,
                   total_amount, total_cost,
                   total_profit, date, notes
6. bill_items    → id, bill_id(FK),
                   product_id(FK), quantity,
                   unit_price, total_price
7. users         → id, username, password_hash,
                   full_name, role, is_active,
                   last_login

USER ROLES:
owner   → Full access to all 7 pages
manager → Products, Sales, Expenses,
          Billing, Dashboard
staff   → Dashboard, Sales, Billing only

IMPORTANT NOTES:
1. Frontend must be accessed via Flask URL:
   http://localhost:5000/pagename.html
   NOT via file:// — CORS will block it!

2. Tokens are stored in memory (dict)
   in auth.py — they reset when server
   restarts — users must re-login

3. ML models need minimum 2 days of
   sales data to give meaningful results

4. Groq free tier limit:
   1000 requests/day, 30/minute

   DB Browser for SQLite
